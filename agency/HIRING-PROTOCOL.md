# Протокол найма ИИ-агента (Agency Hiring Protocol)

> Запускается на шаге «Исполнитель» в Ритуале согласования.
> Цель: подобрать профильного агента под роль задачи или использовать модель по умолчанию.

---

## Алгоритм (4 шага)

### Шаг 1 — Определи роль задачи

Задай себе один вопрос:
> **«Кто должен выполнить эту задачу, если бы это был человек?»**

Примеры:
- «Написать регламент для бариста» → **регламентатор / методолог**
- «Создать Telegram-бота» → **разработчик**
- «Обновить ГПХ» → **HR-специалист**
- «Письмо архитектору» → **архитектор / документовед**
- «Починить скрипт агента» → **инженер среды**
- «Спланировать неделю» → **стратег**

### Шаг 2 — Проверь реестр

Открой `agency/REGISTRY.md` → найди агента с подходящей ролью.

```
Роль задачи          →  Агент в реестре?
─────────────────────────────────────────
Разработчик          →  Code Engineer       ✅
Стратег              →  Strategist          ✅
HR / кадры           →  HR Specialist       ✅
Инженер среды        →  Environment Engineer ✅
Аналитик кофейни     →  VK Coffee Analyst   ✅
Архитектор парка     →  Park Architect      ✅
Копирайтер           →  —                   ❌
Финансист            →  —                   ❌
Маркетолог           →  —                   ❌
```

### Шаг 3 — Нанять или использовать по умолчанию

**Если агент найден:**
```
→ Объявить: «Нанимаю [Имя агента]»
→ Прочитать agency/agents/{slug}.md
→ Загрузить контекст агента (Pack, инструменты, контракт)
→ Работать в роли агента
```

**Если агент НЕ найден:**
```
→ Объявить: «Работаю моделью по умолчанию ([Sonnet/Haiku/Opus])»
→ Зафиксировать: какая роль была нужна (для расширения реестра)
→ Работать без специализации
```

### Шаг 4 — Зафиксировать пробел (если агента нет)

Если подходящего агента не нашлось → добавить в `agency/GAPS.md`:

```markdown
- [2026-03-26] Задача: [описание] → Роль нужна: [роль] → Агент: отсутствует
```

Это позволит понять, каких агентов нанять следующими.

---

## Формула объявления в Ритуале

```
Модель: [Sonnet / Haiku / Opus]
Роль задачи: [что нужно]
Агент: [Имя] из реестра / модель по умолчанию
```

**Пример с агентом:**
> Модель: Sonnet. Роль: разработчик бота. Агент: **Code Engineer**. Загружаю контекст.

**Пример без агента:**
> Модель: Sonnet. Роль: копирайтер для Instagram. Агент: **нет в реестре → модель по умолчанию**. Фиксирую пробел.

---

## Правило расширения реестра

> Если одна и та же роль встречается **3+ раза** в GAPS.md → создать нового агента.

Алгоритм создания:
1. Определить домен и специализацию
2. Создать `agency/agents/{slug}.md` по шаблону
3. Добавить в `REGISTRY.md`
4. Убрать из GAPS.md

---

## Шаблон карточки агента

```markdown
---
agent-id: {slug}
role: {Название роли}
domain: {Pack / репо / область}
model-default: sonnet
version: 1.0
created: {YYYY-MM-DD}
---

# Агент: {Название} ({Русское описание})

## Специализация
[Что умеет лучше обычного Claude]

## Контекст при найме
[Что прочитать перед началом работы]

## Инструменты
[Что использует]

## Контракт
**Делает:** ...
**Не делает:** ...

## Нанимать когда
[Конкретные триггеры]

## Не нанимать когда
[Граница компетенции]
```

*Протокол обновляется при изменении механизма найма.*
