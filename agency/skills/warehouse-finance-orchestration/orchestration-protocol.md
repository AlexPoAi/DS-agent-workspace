# Скилл: Оркестрация обработки отчётов — один trigger, два агента, четыре этапа

## Назначение

Единый управляемый протокол для обработки отчётов Жанны обоими агентами **без конфликтов на файлах**.

## Запуск

**Команда:** `/warehouse-process` или "обработать отчеты"

## Проблема, которую решает

Жанна кидает файлы в Drive → Google Drive папка "Новые документы"

**Наивный подход (ломается):**
- Кладовщик берёт файлы, обрабатывает, ДВИЖЕТ в "Обработано"
- Экономист тоже хочет те же файлы — но их уже нет в "Новые документы"!
- Оба агента работают независимо, нет координации

**Правильный подход (этот скилл):**
- Один оркестратор управляет всем
- Оба агента читают из **локального хранилища** (не из Drive)
- Файлы на Drive трогаются ТОЛЬКО после обработки обоих
- Чёткие этапы: Intake → Кладовщик → Экономист → Архив

---

## Архитектура: 4 этапа (ВЫПОЛНЯЮТСЯ ПОСЛЕДОВАТЕЛЬНО)

### Этап 0: INTAKE (скачивание из Drive)

**Что делает:**
- Скачивает файлы из Google Drive папка "Отчёты для бота/Новые документы"
- Сохраняет их в локальное хранилище: `VK-offee/knowledge-base/Отчёты для бота/Новые документы/`
- **Важно:** файлы на Drive остаются в "Новые документы" (НЕ двигаем пока)

**Скрипт:**
```bash
cd ~/Github/VK-offee
python3 .github/scripts/sync-google-sheets.py
```

**Проверка:**
```bash
ls -la ~/Github/VK-offee/knowledge-base/Отчёты\ для\ бота/Новые\ документы/ | head -20
```

**Выход этапа:** файлы локально, готовы к обработке обоими агентами

---

### Этап 1: КЛАДОВЩИК (обработка в PACK-warehouse)

**Что делает:**
- Читает файлы из **локального хранилища** (не с Drive!)
- Обрабатывает: остатки, продажи, ABC, накладные
- Пишет результаты: WH.CARD.*, WH.SESSION.001, WH.REPORT.002

**❌ ВАЖНО: Кладовщик НЕ ТРОГАЕТ папки Drive и локальные файлы**
- Оставляет все файлы в "Новые документы" (и на Drive, и локально)
- Файлы должны быть доступны для экономиста
- Архивизацию делает только экономист (Этап 3)

**Скрипт:**
```bash
cd ~/Github/VK-offee
bash PACK-warehouse/tools/warehouse_full_loop.sh
```

**Важно:**
- Скрипт должен ТОЛЬКО читать, НЕ двигать файлы на Drive
- Если warehouse_full_loop.sh движет файлы сам → нужно его обновить

**Проверка:**
```bash
grep -i "processed\|обработано" ~/Github/VK-offee/PACK-warehouse/tools/warehouse_full_loop.sh
```
Если есть перемещение → закомментировать эту строку!

**Выход этапа:** WH.SESSION.001 обновлён, рекомендации по закупке готовы

---

### Этап 2: ЭКОНОМИСТ (обработка в DS-finance-private)

**Что делает:**
- Читает файлы из **локального хранилища** (НЕ с Drive!)
- Обрабатывает: Продажи, Выручка, Накладные, Каталог
- Считает метрики: выручка по точкам, маржа, динамика, аномалии
- Пишет Finance View в `DS-finance-private/business-finance/views/finance-view-YYYY-MM-DD.md`
- **Не трогает** файлы (ни на Drive, ни локально)

**Скрипт:**
Финансовый агент запускается через coordinator:
```bash
# Это исполняет Claude при команде "обработай отчеты"
# Claude вызывает: DS-agent-workspace/agency/agents/vk-finance-analyst.md
# Который вызывает skill: agency/skills/vk-finance-analyst/finance-view-builder.md
```

**Важные ограничения для скилла:**
- Читать ТОЛЬКО из локального хранилища
- НЕ скачивать напрямую с Drive (файлы уже скачаны в Этап 0)
- НЕ трогать локальные файлы (не удалять, не перемещать)

**Выход этапа:** Finance View создан, готов для financial-consultant

---

### Этап 3: АРХИВ (только экономист перемещает файлы)

**Что делает:**
- ✅ **ЭКОНОМИСТ перемещает** файлы на Drive: "Новые документы" → "Обработано"
- ❌ **Кладовщик уже закончился** и не трогает папки
- Обновляет реестры обоих контуров
- Отправляет Telegram digest (оба результата: кладовщик + экономист)

**Логика:**
1. Кладовщик прочитал своё, оставил файлы в "Новые документы"
2. Экономист прочитал своё из тех же "Новые документы"
3. Теперь экономист (и только он!) архивирует всё в "Обработано"

**Скрипт (выполняет экономист ПОСЛЕ Finance View создан):**

```bash
# Перемещение на Drive (ТОЛЬКО экономист это делает)
python3 <<'EOF'
from google_drive_parser import GoogleDriveAPI

api = GoogleDriveAPI()

# Переместить ВСЕ файлы из inbox в processed (даже те что экономист сам не читал)
api.move_files(
    source_folder_id="1Jg1Zgj2_ueTV6-NQamAU8XCPalFNhO8W",  # Новые документы
    target_folder_id="1WanukzWeuqJgUQ7N8YkG9oC23-DIRGjT"   # Обработано
)

print("✅ Все файлы перемещены в Обработано")
EOF

# Обновить реестры обоих контуров
# WH.REGISTRY.001 в PACK-warehouse
# финансовый реестр в DS-finance-private/

# Telegram digest (оба результата вместе)
curl -X POST https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage \
  -d chat_id=${CHAT_ID} \
  -d text="✅ Цикл завершен.
  
📦 Кладовщик: WH.SESSION.001 готов (рекомендации по закупке)
💰 Экономист: Finance View готов (маржа, выручка по точкам)

Файлы перемещены в Обработано"
```

**Проверка:**
```bash
# Убедиться что в "Новые документы" ничего не осталось
ls /Users/alexander/Github/VK-offee/knowledge-base/Отчёты\ для\ бота/Новые\ документы/ | wc -l
# Должно быть 0 файлов
```

**Выход этапа:** Цикл завершен. Готово к следующей инвентаризации через 2 недели

## Последовательность: СТРОГО В ПОРЯДКЕ

```
TRIGGER: /warehouse-process
   ↓
[Этап 0] INTAKE — скачать файлы с Drive в local
   (если ошибка → СТОП, не продолжать)
   ↓
[Этап 1] КЛАДОВЩИК — обработать, пишет в PACK-warehouse
   (Кладовщик: "я обработал, готово")
   ↓
[Этап 2] ЭКОНОМИСТ — обработать, пишет в DS-finance-private
   (Экономист: "я обработал, готово")
   ↓
[Этап 3] АРХИВ — переместить на Drive, отправить digest
   ✅ Цикл завершен
```

**НИКОГДА не пропускать этап, никогда не менять порядок!**

## Нормализация (ЕДИНАЯ НА ДВОИХ)

**Точки:** **ОБЯЗАТЕЛЬНО** одинаковые

Из `drive-folders-vkoffee.yaml`:
```yaml
point_slugs:
  тургенева: turgeneva
  карьерный: lugovaya
  самокиша: samokisha
```

**Маржа:** разные типы, но чётко помечены

| Агент | Маржа | Источник | Название |
|-------|-------|---------|---------|
| Кладовщик | Расчётная | Каталог (Саби цены) | `catalog_margin` |
| Экономист | Фактическая | Накладные (реальный COGS) | `actual_margin` |

**Важно:** не должны совпадать, это нормально. Просто помечать ясно.

## Файлы конфигурации

**Canonical Drive Map:**
- Путь: `DS-agent-workspace/agency/config/drive-folders-vkoffee.yaml`
- Содержит: folder IDs, point slugs, нормализация
- Обновляется один раз, используется всеми агентами

**Структура:**
```yaml
---
# Единый источник истины для Drive folder IDs и нормализации
drive_folders:
  root: "1oo1j86l7hGZ-E1HIbAApc3PdCA3o80GX"
  inbox: "1Jg1Zgj2_ueTV6-NQamAU8XCPalFNhO8W"
  processed: "1WanukzWeuqJgUQ7N8YkG9oC23-DIRGjT"

point_slugs:
  тургенева: turgeneva
  карьерный: lugovaya
  самокиша: samokisha

margin_types:
  catalog_margin:
    source: каталог
    used_by: warehouse-demand-analyst
    meaning: расчётная маржа по цене закупки
  actual_margin:
    source: накладные (COGS)
    used_by: vk-finance-analyst
    meaning: фактическая маржа по реальной себестоимости
```
