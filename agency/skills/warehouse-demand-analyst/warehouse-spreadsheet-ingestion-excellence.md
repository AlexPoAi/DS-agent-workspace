# Warehouse Spreadsheet Ingestion Excellence

## Цель

Сделать чтение `xlsx/xls/csv/google sheets` базовой сильной стороной кладовщика.
Если файл прислан из операционного мира, агент обязан дочитать его до реального
data-range и не потерять строки из-за плохого режима чтения или неверно выбранного листа.

## Обязательные правила

1. Перед разбором таблицы агент обязан определить:
- сколько в workbook листов;
- какой у листа `dimension`, `max_row`, `max_col`;
- где начинается реальная таблица;
- есть ли merged headers, service rows, totals, пустые визуальные отступы.

2. Агенту запрещено считать безопасным по умолчанию:
- первый лист;
- первый непустой лист;
- `read_only=True`;
- первые 10-20 строк;
- `csv`, если он не сверен с исходным workbook.

3. Агент обязан проверять качество materialization:
- совпадает ли фактическое число строк в локальном CSV с исходным листом;
- не срезан ли data-range;
- не потеряны ли скрытые/дальние строки;
- не исчезла ли таблица после экспорта.

4. Если один способ чтения режет таблицу, агент обязан переключиться на safer method:
- `openpyxl normal mode` вместо `read_only`, если range обрезается;
- проверка нескольких листов;
- sheet-specific parsing;
- прямое чтение workbook вместо доверия промежуточному CSV.

## SOTA-подход для warehouse

### Метод 1 — Workbook structure first
- открыть workbook;
- собрать метаданные по каждому листу;
- найти вероятный data-sheet;
- только потом читать строки.

### Метод 2 — Data-range validation
- сверить `calculate_dimension`, `max_row/max_col`;
- проверить, что после materialization число строк не схлопнулось;
- если локальный range подозрительно мал, считать это ingestion defect.

### Метод 3 — Header detection, не hardcode
- искать header row по смысловым колонкам, а не по номеру строки;
- уметь переживать двухэтажные заголовки и сервисные строки сверху.

### Метод 4 — Dual-path verification
- для критичных файлов сверять минимум двумя сигналами:
  - workbook live read;
  - локальная materialization.

Если результаты расходятся, агент не имеет права делать молчаливый вывод.

## Quality gate

- [ ] Data-range подтверждён.
- [ ] Header row найден осмысленно.
- [ ] Потерянных строк нет.
- [ ] Лист выбран осознанно.
- [ ] Если есть сомнение, поднят `manual review` с root-cause.
