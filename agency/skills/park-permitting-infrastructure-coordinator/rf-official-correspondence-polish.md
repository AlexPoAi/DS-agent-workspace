# Skill: RF Official Correspondence Polish

## Зачем нужен

Этот skill нужен, чтобы Park-agent не останавливался на “хорошем черновике”,
а доводил официальное письмо до реально готового документа по стандарту
деловой переписки РФ.

## Общий discipline gate

Применять `agency/skills/common/codex-karpathy-discipline.md`.

Для polishing-задач это значит:
- править только официальный вид, реквизиты, ясность и готовность к подаче;
- не менять смысл обращения без отдельного согласования;
- не добавлять юридические утверждения, если они не подтверждены источником;
- проверять финальный документ через readback/рендер, если формат это позволяет.

## Когда запускать

- адресат уже выбран;
- предмет письма уже определён;
- документ надо довести до `clean-ready / print-ready / ready-to-send`.

## Что проверить обязательно

### 1. Шапка и адресный блок

- шапка расположена по правому краю;
- адресат указан официально и без сокращений, где это важно;
- отправитель указан полно.

### 2. Реквизиты

- нет пустых `[ФИО]`, `[адрес]`, `[телефон]`, `[email]`;
- реквизиты совпадают с каноническим источником;
- дата и подпись не забыты.

### 3. Предмет обращения

- предмет узкий и проверяемый;
- вопросы пронумерованы;
- ясно, какой именно ответ ожидается.

### 4. Anti-brush-off hardening

- документ должен по умолчанию быть усилен против формальной отписки;
- надо просить либо ответ по существу, либо письменное указание компетентного органа;
- если адресат некомпетентен, в письме должен быть запрос на переадресацию по подведомственности или на точное указание адресата;
- не оставлять удобный сценарий ответа в стиле “это не к нам” без следующего шага.

### 5. Чистота документа

- нет черновых пометок;
- нет техничекских хвостов;
- приложения оформлены;
- документ выглядит готовым к печати и подаче.

### 6. Пагинация

- если документ по смыслу должен помещаться на один лист, это нужно проверить;
- подписной блок не должен уезжать на вторую страницу;
- если подпись уехала, это не `print-ready`.

## Жёсткие запреты

- Не оставлять плейсхолдеры.
- Не называть документ финальным, если он ещё не вычитан по шапке и реквизитам.
- Не называть документ `print-ready`, если подпись уехала на вторую страницу.
- Не выпускать документ в `print-ready / ready-to-send`, если он не усилен против формальной отписки.
- Не подменять этим skill-ом юридическое заключение.

## Truthful output

После skill документ должен иметь один из статусов:

- `clean-ready`
- `print-ready`
- `ready-to-send`

Если есть хотя бы один незаполненный реквизит, это не выше `draft`.
